DFD Level 2 Details: Order Fulfillment (3.0)

1. Sub-Processes

IDProcess NameFunction
3.1Check StockValidates that requested book quantities are available in D2 inventory.
3.2Calculate TotalSums up item prices and applies any taxes or shipping costs.
3.3Pay ProcessInitiates transaction with the Payment Gateway (PG) and awaits authorization.
3.4Create RecordSaves the final order and generates the customer invoice in D3.
3.5Update StockFinalizes the decrement of the book count in D2.

2. Involved Data Stores

StoreUsage
D2 (Books)Used for inventory verification (read) and quantity deduction (write).
D3 (Orders)Used to store the transaction snapshot and invoice generation.

3. Flow Breakdown

  • User → P3.1: Sends current cart content for validation.
  • P3.3 → PG: Request for secure payment processing (Authorized/Denied).
  • P3.4 → User: Returns a digital receipt (Invoice) on success.
